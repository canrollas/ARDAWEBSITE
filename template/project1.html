The PMS consists of both hardware and software design for healthcare applications. Its purpose is to allow patient monitoring to be done remotely. Remote monitoring of patients avoids the spread of contagious diseases among healthcare workers, especially in the height of Covid-19 pandemic.



The system measures heart rate, blood oxygenation percentage, and body temperature of patients . All sensors were designed from scratch, specifically for this project, to both decrease the dependency of 3rd party hardware, and to decrease the overall project cost. Measured data are then fed into an ESP32 based board (Firebeetle ESP32-WROOM) to be processed and uploaded to an online server dedicated to holding patient data. Lastly, an API grabs the patient data and displays it on a mobile application also developed specifically for the project too. This way, healthcare workers can keep track of all their patients' data remotely, from their mobile phones.
